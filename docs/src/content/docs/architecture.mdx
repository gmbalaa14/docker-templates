## Cross-Template Network Map

Two shared bridge networks connect templates that need to talk to each other. All other services are network-isolated by default.

**Key points:**
- Services are isolated by default. They only join `proxy-net` when you uncomment the NPM opt-in block at the bottom of their `docker-compose.yml`.
- `kafka-ui-net` is always joined by Kafka clusters — the UI cannot discover brokers without it.
- Two separate `proxy-net` and isolated entries for the same service represent the two operating modes (with or without NPM integration).

**Production guidance:**
- In production, expose only NPM on ports 80/443. Keep all service ports bound to `127.0.0.1` or unexposed.
- For Kafka, bind external listeners to a specific host IP rather than `0.0.0.0`.
- Use named volumes (already the default) so data persists across `docker compose down`.

## Shared Networks

| Network | Created by | Joined by | Purpose |
|---|---|---|---|
| `proxy-net` | You (once, manually) | NPM (always), any service that opts in | SSL termination and reverse proxying |
| `kafka-ui-net` | You (once, manually) | Kafka UI, Kafka clusters | Broker discovery from the UI |

```bash
# Create both networks once on your host before starting any service
docker network create proxy-net
docker network create kafka-ui-net
```

For a detailed explanation of how these networks work and how to opt services in, see [Networking](/networking/).
