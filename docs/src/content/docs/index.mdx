## What is this?

A curated collection of self-contained Docker Compose configurations for self-hosted services. Every template is:

- **Fully parameterised** via `.env` files — no secrets hardcoded anywhere
- **Compatible** with Docker CLI, [Portainer](https://www.portainer.io/), and Docker Desktop on Mac and Windows
- **Convention-enforced** — validated by CI on every commit (YAML lint, port conflicts, security scan)
- **Production-oriented** — named volumes, `restart: unless-stopped`, security warnings included

## Quick Start

```bash
# 1. Clone the repository
git clone https://github.com/gmbalaa14/docker-templates.git
cd docker-templates

# 2. Pick a template and navigate to it
cd <template-name>

# 3. Copy and fill in the environment file
cp .env.example .env
# Edit .env with your values

# 4. Start the service
docker compose up -d
```

Nginx Proxy Manager and the Kafka templates also require a one-time shared network — see [Prerequisites](./prerequisites/#shared-network-setup) for which templates need one and the exact command. Most templates need no extra setup beyond the steps above.
